What is Web API?
-> Web API is a set of rules that allows communication between client and server using HTTP or HTTPS for sending requests and receiving responses.
-> they are some operation like a PUT,POST,GET,DELETE like  that operation are perform using the Web API?

Explain Web API fundamentals?
->
Client-Server Communication – Web API connects frontend (client) and backend (server).
Uses HTTP/HTTPS – Communication happens using standard web protocols.
Endpoints (URLs) – Specific paths like /api/users used to access data.

HTTP Methods –
GET → Read data
POST → Create data
PUT → Update data
DELETE → Remove data

Request & Response – Client sends request, server sends back response (usually in JSON).
Stateless – Each request is independent; server doesn’t remember past requests.
Status Codes – Shows result of the request (e.g., 200 OK, 404 Not Found).
Authentication – Secures API using tokens or keys.

✅ HTTP Status Code Ranges:
200–299 → Success
→ The request was successful (e.g., data fetched, created, updated).

300–399 → Redirection
→ The resource has moved or requires redirection.

400–499 → Client Error
→ The request is incorrect or unauthorized (e.g., 404 Not Found, 401 Unauthorized).

500–599 → Server Error
→ The server failed to process the request (e.g., 500 Internal Server Error).
-------------------------------------------------------------------------------
| Code    | Meaning               | Description                               |
| ------- | --------------------- | ----------------------------------------- |
| **200** | OK                    | Request successful                        |
| **201** | Created               | New resource successfully created         |
| **204** | No Content            | Success, but no data to return            |
| **400** | Bad Request           | Invalid request from client               |
| **401** | Unauthorized          | Authentication required                   |
| **403** | Forbidden             | Access denied                             |
| **404** | Not Found             | Resource not found                        |
| **409** | Conflict              | Conflict in the request (e.g., duplicate) |
| **500** | Internal Server Error | Server error while processing request     |
| **503** | Service Unavailable   | Server is down or overloaded              |
-------------------------------------------------------------------------------

What is REST API & There Principal?
->REST API (Representational State Transfer API) is a way of communication between client and server using HTTP or HTTPS, where operations like GET, POST, PUT, and DELETE are used to perform actions on resources.

And There  Principal:-

Sure! Here are the **REST API principles in one line each**:

1. **Stateless** – Each request is independent; no data stored on the server.
2. **Client-Server** – Client and server are separate and communicate via API.
3. **Uniform Interface** – Use a consistent structure for API requests and responses.
4. **Resource-Based** – Everything is treated as a resource accessed via URL.
5. **HTTP Methods** – Use standard methods like GET, POST, PUT, DELETE.
6. **Cacheable** – Responses can be cached to improve performance.

Explain the concept of debounce and throttle. When would you use them?
->
🔸 Debounce is used to delay the function call until the user stops typing. For example, in a search input, it will wait 1 second after the user stops typing and then send only one API request.

🔸 Throttle is used when you want to limit how many times a function is called. For example, if a user clicks the login button many times, throttle ensures that the API is only called once in a few seconds, avoiding server overload.

What is the difference between shallow copy and deep copy?
🔹 A shallow copy copies only the top-level data. If the object has nested objects, those are still linked (reference).
🔹 A deep copy means it copies everything — even the nested objects — so changes in the copied object do not affect the original.
