14.What is normalization?
->Normalization is the process of storing data in a database in a proper way to ensure uniqueness, maintain data integrity, and avoid duplication.
-----------------------------------------------------------------------------------------------------------------
15.What are the types of normalization?
->The main normal forms used in database normalization are:

	1st Normal Form (1NF) – Ensures each column contains atomic values (no multiple values in one column) and each record is unique.

	2nd Normal Form (2NF) – Achieves 1NF and removes partial dependency (non-key attributes depend on the whole primary key).

	3rd Normal Form (3NF) – Achieves 2NF and removes transitive dependency (non-key attributes shouldn’t depend on other non-key attributes).

53.What are the ACID properties?

->✅ Real-Time ACID Properties (Using Your Words, Improved):

Atomicity :- 
When data is transferred from User A to User B, the full process must complete.
If any step fails, the entire transaction is cancelled — no partial changes happen.

Consistency :- 
After the transaction, the data must stay in a valid state.
There should be no data loss or mismatch during the operation, even if it takes time.

Isolation :- 
Each user's transaction is separate and isolated.
One user’s request should not affect another user’s request, even if they run at the same time.

Durability :- 
Once the entire process is successfully completed, the data is saved permanently in the database — even if the system crashes after that.

->Non-Clustered Index means the data is not stored in sorted order, but the index stores sorted references (pointers) to the actual data — like the index page of a book, which helps you find the exact page quickly.

58.How can you copy data from one table to another?
->To copy data from one table to another, we use the INSERT INTO + SELECT command.
The process is:
First, we write the INSERT query for the target (new) table.
Inside it, we write a SELECT subquery from the source table.
The SELECT query runs first, and its output is inserted into the other table.

INSERT INTO new_table (column1, column2, ...)
SELECT column1, column2, ...
FROM old_table;
